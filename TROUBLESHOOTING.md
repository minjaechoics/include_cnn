# 🔧 문제 해결 가이드

"모델 로드 실패" 및 기타 문제 해결 방법입니다.

## 🚨 "모델 로드 실패" 오류

### 원인
이 오류는 다음 중 하나 때문입니다:
1. **인터넷 연결 문제** - MNIST 데이터 다운로드 실패
2. **브라우저 설정** - IndexedDB 또는 CORS 차단
3. **방화벽/보안** - storage.googleapis.com 차단

### ✅ 해결 방법

#### 방법 1: 그냥 기다리기 (권장)
```
"데모 모드로 시작" 메시지가 뜨면
→ 2초 후 자동으로 진행됩니다
→ 랜덤 가중치로 작동 (정확도 낮지만 시각화는 정상)
```

#### 방법 2: 페이지 새로고침
```
F5 또는 Ctrl+R (Mac: Cmd+R)
→ 다시 MNIST 데이터 다운로드 시도
```

#### 방법 3: 브라우저 캐시 삭제
```
Chrome: Ctrl+Shift+Delete
→ "쿠키 및 기타 사이트 데이터" 체크
→ "캐시된 이미지 및 파일" 체크
→ 삭제 후 페이지 재접속
```

#### 방법 4: 다른 브라우저 사용
```
Chrome → Firefox
또는
Firefox → Chrome
또는
Edge 사용
```

- 스크롤해서 전체 내용 확인

## 🐌 로딩이 너무 느려요

### 첫 방문 시 (정상)
```
1. "MNIST 데이터 다운로드 중..." - 10-30초
2. "모델 학습 중..." - 30-60초
3. "모델 저장 중..." - 5초

총 1-2분 소요 (정상)
```

### 두 번째 방문부터
```
"저장된 모델 로드 완료!" - 2-3초
→ 매우 빠름!
```

### 너무 느리면
✅ **해결:**
- 인터넷 속도 확인
- 다른 탭 닫기 (메모리 확보)
- 시크릿 모드에서 테스트

## 🎯 예측이 이상해요

### "숫자를 잘못 맞춰요"
이것은 **정상**입니다:

#### 첫 방문 (학습 전)
- 랜덤 가중치 = 10% 정확도 (거의 무작위)
- **해결:** 1-2분 기다려서 학습 완료

#### 학습 완료 후
- 85-90% 정확도
- 여전히 가끔 틀림 (이것도 정상!)

### 항상 틀리는 경우
✅ **체크리스트:**
- [ ] 숫자를 크게 그렸나요?
- [ ] 숫자가 캔버스 중앙에 있나요?
- [ ] 너무 작게 그리지 않았나요?
- [ ] 학습이 완료되었나요? ("모델 준비 완료!" 메시지 확인)

### 더 정확하게 만들기
```javascript
// app.js 파일에서 수정
const EPOCHS = 3;  // → 5로 증가
const BATCHES_PER_EPOCH = 100;  // → 200으로 증가

// 단, 학습 시간도 2배 증가
```

## 💾 저장이 안 돼요

### "모델 저장 실패"
✅ **원인 및 해결:**

**시크릿/프라이빗 모드**
→ IndexedDB 차단됨
→ 일반 모드에서 사용

**저장 공간 부족**
→ 브라우저 데이터 삭제
→ 디스크 공간 확보

**브라우저 설정**
→ 설정 → 개인정보 → 사이트 데이터 허용

## 🌐 GitHub Pages 배포 문제

### "404 Not Found"
✅ **체크리스트:**
- [ ] Settings → Pages에서 활성화했나요?
- [ ] Branch가 `main`으로 설정되었나요?
- [ ] `index.html` 파일명이 정확한가요? (대소문자 구분)
- [ ] 1-3분 기다렸나요?

### "파일은 보이는데 작동 안 함"
✅ **확인:**
- [ ] `app.js` 파일도 업로드했나요?
- [ ] `style.css` 파일도 업로드했나요?
- [ ] F12 → Console에서 에러 확인

### "빈 화면만 보여요"
✅ **해결:**
1. F12 (개발자 도구) 열기
2. Console 탭 확인
3. 빨간 에러 메시지 확인
4. 에러 메시지를 Issues에 올리기

## ❓ 자주 묻는 질문

### Q: 첫 방문이 너무 느려요
**A:** 정상입니다! MNIST 데이터 다운로드(10MB) + 학습이 진행됩니다. 다음 방문부터는 빠릅니다.

### Q: 왜 가끔 틀리나요?
**A:** MNIST 테스트 정확도가 85-90%라서 10-15%는 틀립니다. 완벽한 모델은 없어요!

### Q: 오프라인에서 작동하나요?
**A:** 한 번 방문 후에는 모델이 브라우저에 저장되어 오프라인 작동 가능합니다.

### Q: 다른 숫자도 인식하나요?
**A:** MNIST는 0-9 손글씨 숫자만 학습했습니다. 다른 것은 인식 못 해요.

### Q: 한글도 인식하나요?
**A:** 아니요, 이 모델은 숫자 전용입니다.

## 🆘 그래도 안 되면

```
F12 → Console 탭

```

### 2단계: GitHub Issues
```
https://github.com/[저장소]/issues
→ New Issue
→ 에러 스크린샷 첨부
→ 브라우저 종류/버전 명시
```

### 3단계: 임시 해결
```
랜덤 가중치로라도 시각화는 작동합니다
→ 데모용으로는 충분!
```
